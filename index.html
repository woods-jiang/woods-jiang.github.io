《当幸福来敲门》观后感

By Woods Jiang

发表于 2015-06-22

昨天是父亲节，看到很多推荐有关父亲相关的电影，其中《当幸福来敲门》在很多推荐里面都是排到第一位的，想想自己对其中的一些情节印象已经有些模糊了，于是又拿出来看了一遍。

看完电影之后，我让我感触最深的并不是什么父爱，虽然这里面确实有很多表现父爱的情节，比如说他们第一次在地铁下面的厕所里面睡觉时，半夜有人来敲门，父亲抱着儿子的头，然后痛哭的哭了，看到这儿的时候我心里也为之一振。

而让我反思的是：一个人想要做成一件有点难度的事情，有三样东西特别重要：智商、坚韧的性格、好运气

首先看我们的男主，为了找到股票经纪人的工作，经常去跟那个高管打招呼，但是别人都不理他，直到后来他把那个魔方用很快的速度拼好之后，才开始关注到他。要知道魔方是1974年发明的，而真正开始推广是在1980年，也就是男主买魔方的那一年，当时还没有什么公式，男主凭借自己对数字的敏感和聪明的头脑，买回来当天就完成了现在拼魔方常规步骤的前两步。要知道，当时电视里面说，一个人要想完全的拼好它，一个小时都不够的（这还得是经常玩的高手才行吧），那个股票经纪人甚至一个面都拼不好，而男主呢，在坐车的那么一会儿时间就拼好了。让那个股票经纪人是非惊叹！感觉他的表情用惊讶都很难形容了。

还有一个侧面证明男主智商高，就是他的数学知识非常好，想想做股票经纪人的，数学知识肯定不简单，后来做试卷还能提前那么久交卷等等侧面，都证明了男主智商高于普通人的。另外，一个高中毕业的学生，居然可以修好那个复杂的仪器，让他最后还能撑一段时间，不然完全没钱的他，怎么还能去做每一分钱工资的股票经纪人实习生呢？

第二点，坚韧的性格，这个不用多说，整部影片都可以感觉到，男主内心的强大和乐观。而且很多人以为这是唯一重要的因素，如果真是这样，这部片就成了鸡汤片了，不过善于观察的你可能也发现了，这并不是作者唯一要表达的意思。

最后一点，好运气，无疑，男主的运气是非常好的，比如恰好遇到一个喜欢魔方的股票经纪人，而且恰好那天他带着魔方，而且恰好那天遇到他，而且恰好有足够的时间让他能够拼好魔方。第二个运气好的表现是遇到一个退休基金的负责人，虽然这个人后来并没有跟他谈成，说他是菜鸟，但是后来带他去看美式足球时，男主结交了其他有钱人，这也是他后来成功的一个原因吧。要是没有遇到那个退休基金的负责人，很可能就没那么顺利了。

除了这三点，其实男主的情商也是相当高的，而且还很有幽默感，比如说，在面试实习生时，老板问，我为什么要雇一个光着胳膊的人，男主非常机智的回答道，那么他一定穿了一件好看的裤子。引得面试官哈哈大笑，最后就被录用了。不过我并没有把这一点加入到我的三点感悟中，让它成为我的第四点感悟，因为感觉现在这一点被强调得太多了。而且说话说三点，感觉有力道一点，说的太多看得人会觉得疲倦的。

这也是为什么我喜欢这部片子的原因吧，不是一味的鸡汤电影，什么努力就能成功之类的，它还告诉我们智商、情商、幽默感、好运气也都是非常重要的。

“裸体牛仔”的启示

By Woods Jiang

发表于 2015-06-21

早上看到罗辑思维推送的一篇文章，时代广场的裸体牛仔，讲述的是一个叫罗伯特·伯克的人，大学毕业去好莱坞闯荡，希望成为一名职业模特，但是几年之后混得并不怎么样，后来在朋友的一次恶作剧的调侃下，他脱掉了衣服，在海滩上边走边唱歌，最后这一天他竟然赚了100多元，而且一家电视台采访了他。

后来他便得出结论：要赚到钱，你必须和别人不一样。

这也是文章前半部分的主题吧，虽然这个观点很多人都已经知道了，比如我在上社会计算这门课的时候，老师也反复的说到了，什么二八定律，随大流之类的观点，但是听起来却没有什么感觉，可能就缺乏一个生动的例子吧，比如今天看到的这个故事，很容易让人记住而且进行深入的思考。

不过这样只要与众不同就能够获得成功吗？当然不是，文章中还提到了伯克的另外一个非常重要的品质，可能被很多人忽略掉了，那就是坚持，他坚持17年在时代广场前面抱着吉他唱歌，即使在零下10度的环境中。我觉得，这才是他能够打动别人的所在吧。

如果你只是某一小段时间在时代广场前面裸唱，别人可能觉得你是在炒作，但是像伯克这样，坚持了17年，而且在零下10度这样冷的情况下，还在哪儿裸唱，人们就会竖起大拇指了。

当然，伯克做这些来自于他的其中一个愿望，就是一辈子保持快乐，和让别人快乐。我想这才是他的精神来源吧，如果没有这样的愿望，怎么能够坚持这么久呢？

最后总结三点：

  • 做一个跟别人不一样的人

  • 将一件事情坚持的做下去

  • 让自己保持快乐，并且还能让别人快乐

题目：找到一颗树中两个节点的最近公共祖先，树采用邻接矩阵存储

思路：采用深度优先搜索，用栈记录遍历过程中，从根节点到遍历的当前节点的路径，找到第一个节点之后，栈1就不变了，然后接着前面没有遍历完的继续遍历，不过接下来用栈2来记录，直到找到第二个节点为止。

两个栈分别存放了能够找到两个节点的路径，一次出栈，找到他们相同的那个元素，即为最近公共祖先

[leetcode]Best Time to Buy and Sell Stock IV

By Woods Jiang

发表于 2015-04-25

题目

Say you have an array for which the $i^{th}$ element is the price of a given stock on day $i$.

Design an algorithm to find the maximum profit. You may complete at most k transactions.

Note:
You may not engage in multiple transactions at the same time (ie, you must sell the stock before you buy again).

思路

使用local来保存局部最优解，global来保存全局最优解

状态转移方程如下：
local[i][j] = max(local[i-1][j]+diff, global[i-1][j]+max(diff,0))
global[i][j] = max(global[i-1][j],local[i][j])

local[i][j]：当前达到第i天最多可以交易j次交易，而且最后一次交易在当天卖出，所得到的最大利润，注意，local的第j次交易必须发生在当天
global[i][j]：当前达到第i天最多可以进行j次交易，所得到的最大利润

分析

时间复杂度为$O(n*k)$，空间复杂度为$2O(k)$

[leetcode]Best Time to Buy and Sell Stock III

By Woods Jiang

发表于 2015-04-25

题目

Say you have an array for which the $i^{th}$ element is the price of a given stock on day $i$.

Design an algorithm to find the maximum profit. You may complete at most two transactions.

Note:
You may not engage in multiple transactions at the same time (ie, you must sell the stock before you buy again).

思路

参考上上一篇文章Best Time to Buy and Sell Stock，先前是只交易一次，现在是交易两次，那么我们采取同样的策略，遍历两边就是了

我们使用一个辅助数组maxProfits[0..i..n]来保存第i天卖出，将获得的最大利益。

第一次从左向右遍历，更新maxProfits数组，这样我们就知道了任意一天卖出股票之后获得的收益（只进行一次交易）

第二次从右往左遍历，使用maxPrice保存局部区域的最大值，一旦发现maxPrice - prices[i] + maxProfits[i] > profit，则更新profit，这里，profit就是保存两次交易i获得的最大值。

这里稍微解释一下maxPrice - prices[i] + maxProfits[i]，maxProfits[i]前面已经说过了，是保存第一次交易在第i天完成时，获得的最大利益，而maxPrice - prices[i]则表示第i天买入，maxPrice那天卖出获得的最大利益，两者相加，则表示两次交易获得的最大利益

分析

两次遍历时间复杂度均为$O(n)$，总的时间复杂度还是$O(n)$，空间复杂度为$O(n)$

发表于 2015-04-24

题目

Given a sorted array of integers, find the starting and ending position of a given target value.

Your algorithm’s runtime complexity must be in the order of O(log$n$).

If the target is not found in the array, return [-1, -1].

For example,
Given [5, 7, 7, 8, 8, 10] and target value 8,
return [3, 4].

思路

采用二分查找的思想，首先，找到最左边的等于target的数，如果没有找到就返回[-1,-1]
然后再使用二分查找一次，找到最右边的等于target的数
两次二分查找时间复杂度均为O(log$n$)，所以总的时间复杂度为O(log$n$)

注意体会两个while中的if判断条件的不同

字符串编辑距离

By Woods Jiang

发表于 2015-04-23

编辑距离

编辑距离，又称Levenshtein距离，是指两个字串之间，由一个转成另一个所需的最少编辑操作次数。许可的编辑操作包括将一个字符替换成另一个字符，插入一个字符，删除一个字符。

例如将kitten一字转成sitting：

  1. sitten （k→s）
  2. sittin （e→i）
  3. sitting （→g）

俄罗斯科学家Vladimir Levenshtein在1965年提出这个概念。（来源于维基百科 ）

思路

采用动态规划求解，下面是状态转移方程
if i==0 && j==0, dp[i][j] = 0;
if i==0 && j>0, dp[i][j] = j;
if i>0 && j==0, dp[i][j] = i;
if i>=1 && j>=1, dp[i][j] = min(min(dp[i-1][j],dp[i][j-1])+1,dp[i-1][j-1]+same)
上面的same满足如下条件
if s1[i-1]==s2[j-1], same=0;
else same = 1;
具体实现就比较简单了。
